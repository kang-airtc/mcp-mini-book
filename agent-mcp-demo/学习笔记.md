# MCP 学习笔记

## MCP 是什么

MCP (Model Context Protocol) 是 AI 应用连接外部工具/数据的标准协议。就像 USB-C 统一了设备连接方式，MCP 统一了 AI 与外部系统的连接方式。

---

## 三个核心概念

### 1. Resources (资源) - "这是数据"

**作用**: 给 AI 提供上下文数据，比如文件、数据库记录

**特点**: 
- 通过 URI 标识，如 `file:///project/readme.md`
- 用户主动选择要用哪些数据

```json
// 服务器暴露的资源
{
  "uri": "file:///project/readme.md",
  "name": "README",
  "mimeType": "text/markdown",
  "text": "# 项目介绍\n这是一个示例项目..."
}

// 客户端读取资源
{
  "method": "resources/read",
  "params": { "uri": "file:///project/readme.md" }
}
```

**实际例子**: 
- 代码编辑器的文件列表
- 数据库的表结构信息
- API 文档内容

---

### 2. Tools (工具) - "这是功能"

**作用**: AI 可以调用的功能，比如查询天气、执行计算

**特点**:
- AI 自动发现和调用
- 需要用户授权确认
- 有输入参数和输出结果

```json
// 定义工具
{
  "name": "get_weather",
  "description": "查询指定城市的天气",
  "inputSchema": {
    "type": "object",
    "properties": {
      "city": { "type": "string", "description": "城市名称" }
    },
    "required": ["city"]
  }
}

// AI 调用工具
{
  "method": "tools/call",
  "params": {
    "name": "get_weather",
    "arguments": { "city": "北京" }
  }
}

// 返回结果
{
  "content": [
    { "type": "text", "text": "北京今天晴，25°C" }
  ],
  "isError": false
}
```

**实际例子**:
- 天气查询 API
- 数据库查询
- 文件操作（创建/删除/修改）

---

### 3. Prompts (提示) - "这是模板"

**作用**: 预定义的提示模板，帮助用户快速开始某个任务

**特点**:
- 用户主动选择使用
- 可带参数填充
- 通常显示为斜杠命令

```json
// 定义提示模板
{
  "name": "code_review",
  "title": "代码审查",
  "description": "请 AI 审查代码质量",
  "arguments": [
    { "name": "code", "description": "要审查的代码", "required": true }
  ]
}

// 获取提示内容
{
  "method": "prompts/get",
  "params": {
    "name": "code_review",
    "arguments": { "code": "function add(a,b) { return a+b; }" }
  }
}

// 返回结构化消息
{
  "messages": [
    {
      "role": "user",
      "content": {
        "type": "text", 
        "text": "请审查以下代码：\nfunction add(a,b) { return a+b; }"
      }
    }
  ]
}
```

**实际例子**:
- `/code_review` - 代码审查
- `/explain` - 解释选中代码
- `/refactor` - 重构建议

## 快速对比

| 概念 | 提供方 | 使用方式 | 典型场景 |
|------|--------|----------|----------|
| **Resources** | Server → Client | 用户选择数据 | 文件、数据库、文档 |
| **Tools** | Server → Client | AI 自动调用 | API 调用、计算、查询 |
| **Prompts** | Server → Client | 用户触发模板 | 代码审查、生成模板 |

---

## 完整工作流程示例

```
用户: "帮我查看项目文档并总结一下"

1. AI 发现 Resources:
   - file:///project/readme.md
   - file:///project/docs/api.md

2. 用户选择 readme.md

3. AI 读取内容并总结...

4. 用户: "查询一下今天的天气"

5. AI 发现 Tools:
   - get_weather (查询天气)
   
6. AI 自动调用 get_weather({city: "北京"})

7. 返回结果给用户

8. 用户输入 /code_review (Prompt)

9. AI 加载代码审查模板，分析当前文件
```
